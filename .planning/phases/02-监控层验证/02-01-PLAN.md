  <behavior>
    测试分三组：

    【单元测试组 — WeChatFileHandler 行为，mock task_callback 和 logger】
    - 写入支持扩展名（.jpg .png .gif .bmp .webp .pdf .doc .docx .txt .xlsx）→ task_callback 被调用 1 次
    - 写入不支持扩展名（.exe .mp4 .zip）→ task_callback 不被调用
    - event.is_directory=True → task_callback 不被调用
    - 同一路径连续调用 on_created 两次 → task_callback 仅被调用 1 次（_processed_files 去重）
    - on_created 成功处理后 → logger.info 被调用，消息包含 '检测到新文件'
    - task_callback 抛出异常 → logger.error 被调用，含 '任务触发失败'

    【集成测试组 — 真实 watchdog Observer + tmp_path，Windows-only skip】
    标记：pytest.mark.skipif(sys.platform != 'win32', reason="WeChat monitor is Windows-only")
    - 向 tmp_path/Image/ 写入 test.jpg（>0 字节）→ 5 秒内 task_callback 被调用 1 次
    - 连续写入同一文件路径两次（删除后重写）→ 仅第一次触发（cooldown 内去重）
    - tmp_path 内写入 test.txt → task_callback 不被调用（但 WeChatFileHandler 支持 .txt，需写 .exe 验证非支持类型）
      注：.txt 属于 SUPPORTED_DOC_EXTS，因此用 .exe 作为不支持类型测试

    【WeChatMonitor 单元测试】
    - monitor.start() 当所有 watch_dirs 不存在 → 返回 False，logger.error 被调用
    - monitor.start() 当至少一个 watch_dirs 存在 → 返回 True，observer.start() 被调用
    - monitor.stop() 后 is_running() 返回 False

    Cases（handler 单元测试）:
      FileCreatedEvent(".jpg", is_directory=False) + 文件存在 → callback called once
      FileCreatedEvent(".exe", is_directory=False) → callback not called
      FileCreatedEvent(".jpg", is_directory=True) → callback not called
      FileCreatedEvent(".jpg") called twice with same path → callback called once
  </behavior>

    6. observer fixture：在 conftest 或测试内 yield observer，teardown 调用 observer.stop() + observer.join()
    7. WeChatMonitor.start() 测试：patch Observer，传入存在/不存在的 tmp_path

    注意：WeChatFileHandler.on_created 内有 time.sleep(0.5) 等待文件写入。
