**问题分析：**

现有 agent2_retrieval.py 第 52 行调用 `self.db.search(query=query_text, top_k=self.top_k)`，但 `LanceDBClient`（utils/lancedb_client.py）只提供 `search_similar(text, top_k)` 方法，无 `search()` 方法。此外，LanceDB 返回字段含 `_distance`（L2 距离，越小越相似），但 Agent2 代码和下游期望字段为 `similarity`，需统一转换为 `similarity_score`。

**修复内容（最小化改动）：**

1. 将 `self.db.search(query=query_text, top_k=self.top_k)` 改为 `self.db.search_similar(query_text, self.top_k)`
2. 在 `search_similar` 返回结果后，将每条记录的 `_distance` 字段转换为 `similarity_score`：
   - LanceDB L2 距离转相似度：`similarity_score = max(0.0, 1.0 - _distance)` （sentence-transformers 归一化向量的 L2 距离范围约 0-2，1-distance 得到 0-1 近似相似度）
   - 给每条 case 增加字段：`case["similarity_score"] = round(max(0.0, 1.0 - case.get("_distance", 1.0)), 4)`
3. 修改 `low_similarity_warning` 的判断逻辑：
   - 原来：`len(relevant_cases) < self.top_k // 2`（不依赖阈值）
   - 改为：`avg_similarity < self.similarity_threshold`（与 ROADMAP 规格一致）
4. 修改 `relevant_cases` 的过滤条件：
   - 原来：`case.get("similarity", 0) >= self.similarity_threshold`（字段名错误）
   - 改为：`case.get("similarity_score", 0) >= self.similarity_threshold`
5. result 字典中增加 `cases` 字段（含 similarity_score 的完整 TOP-5 列表，下游 Agent3 使用）：
   - `"cases": similar_cases` （完整列表，含 similarity_score）
   - `"relevant_cases"` 保留（过滤后的高相关案例）

保留其余逻辑不变（_build_query、step2.json 写入、logger 调用）。
