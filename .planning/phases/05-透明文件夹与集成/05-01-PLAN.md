**测试策略：**
- 使用 unittest.mock.patch 分别 mock 五个 Agent 的 run() 方法，返回预设的结构化 dict
- 每个 Agent mock 返回值必须包含该 Agent step_dir 写入 stepN.json 的逻辑 — 由于各 Agent.run() 本身负责写 stepN.json，需让 mock 的 side_effect 实际写入文件（否则 step_dir 为空）
- Orchestrator 初始化时使用 tmp_path 路径（pipeline_dir / conversations_dir / reports_dir）

**mock side_effect 写文件模式（每个 Agent 都要这样处理）：**
```python
def make_agent_run(step_n: int, return_data: dict):
    def side_effect(*args, **kwargs):
        # args[-1] 是 step_dir（Path 对象）
        step_dir = args[-1]
        import json
        (step_dir / f"step{step_n}.json").write_text(
            json.dumps(return_data, ensure_ascii=False), encoding="utf-8"
        )
        return return_data
    return side_effect
```
